# 每日复盘 (Daily Log) Workflow

当用户输入"收工"、"总结"、"今天结束了"或 `/daily-log` 时，按此流程生成工作日志。

---

## Step 1: 回顾当日对话

分析今天的所有对话记录，提取以下信息：
- 完成了哪些功能开发或问题修复
- 遇到了哪些技术难点
- 做出了哪些重要决策

---

## Step 2: 生成结构化日志

按以下格式生成日志内容：

```markdown
# 工作日志 - [YYYY-MM-DD]

## ✅ 今日完成

- [ ] 功能/任务 1
- [ ] 功能/任务 2
- [ ] ...

## 💡 关键难点与解决方案

### 难点 1: [问题描述]
**解决方案**: [如何解决的]

### 难点 2: [问题描述]
**解决方案**: [如何解决的]

## 📋 明日待办 (TODO)

- [ ] 任务 1
- [ ] 任务 2
- [ ] ...

## 📝 备注

[任何需要记录的额外信息，如环境配置变更、依赖更新等]
```

---

## Step 3: 保存日志文件

// turbo
将日志保存到 `docs/` 目录：

```bash
# 文件命名格式: dev-log-YYYY-MM-DD.md
# 例如: docs/dev-log-2026-01-16.md
```

如果当日日志已存在，则追加内容而非覆盖。

---

## Step 4: Git 提交建议

完成日志后，建议用户提交代码：

```bash
git add .
git commit -m "chore: 每日复盘 [YYYY-MM-DD]"
```

---

## 注意事项

1. 日志应当**简洁明了**，每项内容控制在 1-2 行
2. 明日待办应**具体可执行**，避免模糊描述如"继续开发"
3. 如果今天没有明显难点，可省略"关键难点"部分
4. 备注部分仅在有特殊事项时填写
