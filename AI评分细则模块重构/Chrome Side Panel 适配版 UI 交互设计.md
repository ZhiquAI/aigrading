# **Chrome Side Panel 适配版：智能评分 UI 设计**

由于侧边栏宽度受限（类似移动端布局），我们放弃“左右分屏”，采用 **“卡片流 (Card Stream)”** \+ **“切片证据 (Snippet Evidence)”** 的设计模式。

## **1\. 核心布局策略**

* **垂直堆叠 (Vertical Stack):** 所有内容从上到下排列，利用鼠标滚轮。  
* **抽屉式/手风琴 (Accordion):** 得分点默认折叠，点击展开编辑。  
* **切片证据 (Inline Snippets):** 不再展示整张大图，而是将 AI 识别到的 **“那一句答案”** 截图或提取文本，直接显示在对应的规则卡片内部。

### **状态 B: 审核列表 (The Review Stream) —— 核心改动**

这是老师工作的主要界面。

* **顶部悬浮栏 (Sticky Header):** 显示总分统计，以及一个“查看原图”的小按钮（点击弹窗/浮层显示完整图片）。  
* **中间列表 (Scrollable):** 每个得分点是一个卡片。  
* **底部固定栏 (Sticky Footer):** 保存按钮。

## **3\. 具体交互逻辑**

### **3.1 怎么查看原图？(The Source View)**

在侧边栏看大图很痛苦。解决方案：

* **方案 A (Popover):** 顶部有一个小缩略图 "原图附件"。鼠标悬停或点击时，向左侧（主浏览器区域）或在侧边栏上方弹出一个大的浮层显示原图。  
* **方案 B (Snippet First):** 90% 的情况下，老师不需要看整页原图，只需要看**和当前得分点相关的那一句话**。因此，我们优先在卡片里展示 **Crop（切片）**。

### **3.2 如何添加漏掉的规则？**

由于不能在原图上画圈（空间太小），我们改为 **文本添加** 或 **全图模式添加**。

* **操作:** 点击底部的 \[ \+ 添加规则 \]。  
* **辅助:** 弹出一个临时全屏遮罩（覆盖整个 Side Panel），展示完整答案图片，允许老师在上面框选。框选后自动切回列表界面，并填入新规则。

## **5\. 总结：Side Panel 适配要点**

1. **纵向流:** 一切操作上下滑动，避免左右横跳。  
2. **折叠卡片:** 利用有限屏幕，只在需要时展开细节。  
3. **内嵌切片:** 把“证据”直接搬进卡片里，解决无法左右对照的问题。  
4. **全屏弹窗:** 需要复杂操作（如框选原图）时，利用 Overlay 覆盖层暂时获取最大空间。
